Introduçāo¶

A anlise de sentimentos atravs do Processamento de Linguagem Natural (Natural Language Processing NLP), se tornou um tema muito abordado no mundo acadmico e corporativo, muito devido ao crescente nmero de aplicaes prticas, tais como:

  • Identificar se uma avaliao feita em um livro na Amazon foi positiva ou negativa;
  • Identificar sintomas de depresso em pessoas a partir de suas postagens em redes sociais como Facebook ou Twitter;
  • Encontrar potenciais eleitores favorveis a um determinado candidato.

Muitos mtodos e ferramentas tm sido desenvolvidos para a deteco de sentimentos em diversos ambientes e casos. Contudo, grande parte dos recursos para tais anlises esto disponibilizados apenas na lngua inglesa, considerando que esta a lngua predominante no mundo. Entre as diversas ferramentas para tais anlises a API de Natural Language do Google tem se destacado como uma tima opo por possuir suporte a vrios idiomas, entre eles o portugues.

Sobre este estudo¶

Recentemente utilizei pela primeira vez esta API para um trabalho da pos-graduacao. Meu colega de equipe sugeriu utilizar como base para o trabalho, o renomado Dataset do IMDB de reviews de filmes, este dataset originalmente se encontra apenas em ingls mas um usurio do Kaggle (Plataforma para projetos e competies de Data Science) traduziu o dataset para portugus e o dispos na sua pgina pessoal da plataforma (Lus Fred).

Segundo o prprio Lus a traduo foi feita automaticamente, usando uma ferramenta de traduo apropriada que apesar de no citada fez um trabalho muito bom. A ttulo de comparao utilizei outra API do Google (Cloud Translation API) para traduzir algumas da reviews, e pude verificar que os resultados tambm possuam uma tima qualidade e grande similaridade apesar de no serem totalmente iguais.

O trabalho desenvolvido para a ps possua outro foco e utilizou-se apenas dos dados traduzidos em portugus, mas me fez pensar se haveria diferena na classificao do sentimento, utilizando exatamente o mesmo texto apenas com idiomas diferentes.

Neste notebook, iremos verificar e analisar as diferenas e similaridades das duas verses de textos atravs da API de Natural Language do Google.

Amostra¶

O uso da API Natural Language pago e o preo calculado de acordo com o nmero de unidades usadas mensalmente. Cada documento, no nosso caso review enviado para a API para anlise conta como pelo menos uma unidade. Documentos com mais de 1.000 caracteres unicode (incluindo caractere de espao em branco e todos os de marcao, como tags HTML ou XML) so considerados como vrias unidades. Considera-se uma unidade a cada 1.000 caracteres.

Por exemplo, se voc enviar trs solicitaes API Natural Language com 800, 1.500 e 600 caracteres respectivamente, voc ser cobrado por quatro unidades: uma para a primeira solicitao (800), duas para a segunda solicitao (1.500) e uma para a terceira solicitao (600).

Processamento em Paralelo¶

Nosso contedo para anlise foi reduzido consideravelmente mas imagine que quisssemos continuar com o contedo original, isso geraria uma demanda imensa de requisies a API do Google, cada uma com seu prprio tempo de processamento e resposta. No que os servidores poderosssimos de uma empresa como a GOOGLE sejam lentos (Bem longe disso) mas existindo a possibilidade de acelerar esse processo ainda mais porque no nos aproveitarmos dele.

Se fossemos fazer uma iterao normal entre todos os elementos a serem analisados, cada review dentro do nosso loop seria sequencialmente (um a um) enviado a API , analisado e retornado uma resposta, levando um tempo considervel e a que entra o processamento paralelo.

Basicamente processamento em paralelo fazer duas ou mais coisas ao mesmo tempo, isso possvel rodando seu cdigo em diferentes ncleos do computador simultaneamente. A utilizao deste mtodo de processamento possibilita que uma tarefa no tenha que correr de forma sequencial, nos dando a opo de configurar a funo analyze_sentiment(content) para que faa vrias requisies concomitantes a API, utilizando ncleos diferentes consequentemente tendo um ganho em performance.

O parmetro n_jobs seleciona o nmero de processos simultneos que devero ser iniciados e multiprocessing.cpu_count() conta automaticamente o nmero de ncleos do computador para que

def analyze_sentiment(content):
    client = language_v1.LanguageServiceClient()

    # content = 'Your text to analyze, e.g. Hello, world!'

    if isinstance(content, six.binary_type):
        content = content.decode('utf-8')

    type_ = enums.Document.Type.PLAIN_TEXT
    document = {'type': type_, 'content': content}

    response = client.analyze_sentiment(document)
    sentiment = response.document_sentiment
    score = sentiment.score
    magnitude = sentiment.magnitude
    
    return score,magnitude

Avaliando os Resultados¶

A anlise de sentimento da API de Natural Language do Google tenta determinar a atitude geral do documento e representada por valores numricos score e magnitude.

Para classificar corretamente o sentimento de textos, necessrio interpretar esses dois valores. Enquanto o score representa o sentimento geral do contedo (positivo ou negativo), a magnitude indica o contedo emocional presente nele, e esse valor segundo a GOOGLE geralmente proporcional ao tamanho do documento. Resumidamente:

  • Score: indica o range do sentimento, onde -1.0 representa o limite de um sentimento negativo e 1.0 o limite de um sentimento positivo.
  • Magnitude: Indica a fora geral da emoo classificada no score. Diferente do score, a magnitude no normalizada e varia entre os intervalos de 0.0 e +inf.

amostra['sentiment_pt'] = np.select(conditions2, choices, default='null')
